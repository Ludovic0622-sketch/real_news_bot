Reporting a Vulnerability

Nous prenons la sécurité de ce projet très au sérieux. Nous vous encourageons à nous signaler tout problème de sécurité que vous découvrez.

Comment signaler une vulnérabilité

Nous vous demandons de signaler les problèmes de sécurité de manière responsable et en privé via l'une des méthodes suivantes :

Via GitHub Private Security Advisory (Recommandé) :

Allez dans l'onglet Security de ce dépôt.

Cliquez sur Report a vulnerability et remplissez le formulaire.

Par Email :

Si l'option ci-dessus n'est pas disponible, veuillez envoyer un email au responsable du projet à une adresse dédiée (par exemple, [votre-email-de-contact]@example.com).

Veuillez inclure le plus de détails possible pour reproduire le problème : les étapes, la version concernée, et l'impact potentiel.

Notre processus de réponse

Accusé de réception : Nous accuserons réception de votre rapport dans les 48 heures ouvrables.

Mise à jour de l'état : Nous vous fournirons une première mise à jour sur l'état du problème (accepté, refusé ou besoin de plus d'informations) dans les 7 jours suivant l'accusé de réception.

Résolution : Notre objectif est de publier un correctif dans les 30 jours pour les vulnérabilités de gravité élevée, en visant la version supportée la plus ancienne qui ne causerait pas de régression majeure.

Nous vous demandons de ne pas divulguer publiquement la vulnérabilité tant que nous n'avons pas eu l'occasion d'émettre un correctif. Nous créditerons tous les chercheurs en sécurité ayant contribué à améliorer la sécurité du projet.
